This guide provides instructions for installing Passionfruit Microsoft add-ins.

## For Regular Users

If you're a regular user looking to install the Passionfruit add-in for Microsoft applications, follow these steps:

### 1. Open Microsoft Office Application

- Launch your Microsoft Office application (Outlook, Word, Excel, etc.)
- Click on the "Insert" tab in the ribbon

### 2. Access Add-ins

- Click on "Get Add-ins" or "My Add-ins"
- Select "Admin Managed" or "My Add-ins" from the dropdown

### 3. Install Passionfruit

- Search for "Passionfruit" in the search bar
- Click "Add" next to the Passionfruit add-in
- Wait for the installation to complete

### 4. Start Using

- Once installed, you'll find Passionfruit in your add-ins list
- Click on the Passionfruit icon to start using the add-in

### Troubleshooting

- If the add-in doesn't appear, try refreshing your Microsoft application
- Ensure you have the necessary permissions to install add-ins
- Contact your IT administrator if you don't see the add-in in the store

## For Administrators

As an administrator, you'll need to deploy the Passionfruit add-in across your organization. Here's how to do it:

### 1. Access Microsoft 365 Admin Center

- Log in to the [Microsoft 365 Admin Center](https://admin.microsoft.com)
- Navigate to "Settings" > "Integrated apps"

### 2. Add New App

- Click "Add apps"
- Search for "Passionfruit"
- Select the Passionfruit add-in from the results

### 3. Configure Deployment

- Choose deployment options:
  - User Assignment: Select which users or groups should have access
  - Installation Scope: Choose between automatic or optional installation
  - Permissions: Review and approve required permissions

### 4. Deploy the Add-in

- Click "Deploy" to start the deployment process
- Monitor the deployment status in the admin center

### 5. Verify Installation

- Check the deployment status for all users
- Ensure the add-in appears in the specified Microsoft applications

### Troubleshooting

- Check deployment status in the admin center
- Verify user assignments and permissions
- Review Microsoft 365 logs for any deployment issues
- Ensure your organization's policies allow add-in installation
