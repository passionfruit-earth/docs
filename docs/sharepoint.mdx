#### Overview

Connect Passionfruit to SharePoint Online to discover and reuse the right evidence while filling questionnaires. This integration is offered as a managed setup: you request access, and we configure everything with your team using Microsoft Entra ID.

  Available on request. Email `hello@passionfruit.earth` to get started.
</Info>

#### Authentication and Security

The SharePoint integration leverages Microsoft Entra ID for secure authentication and access control. This ensures:

- **Seamless SSO**: If your organisation has configured [Single Sign-On (SSO)](/docs/installation/sso), users can access SharePoint-synced evidence using the same credentials
- **Enterprise security**: Benefit from your existing Microsoft Entra ID security policies, including multi-factor authentication and conditional access rules
- **Secure data access**: All SharePoint access is authenticated through Microsoft Entra ID, ensuring only authorized users can sync and access evidence

#### Purpose

Syncing evidence files with your Knowledge Base keeps your source documents authoritative while making them instantly reusable across Passionfruit.

- **Single source of truth**: Files stay in SharePoint, while Passionfruit indexes them so your team can search, reference, and reuse the latest versions during questionnaires.
- **Always up to date**: Changes at the source are reflected automatically, removing manual re-uploads and stale copies.
- **Governance and analytics**: Centralized visibility for expiry tracking, approvals, ownership, and usage insights.
- **Speed and consistency**: Reuse approved evidence and answers, reducing duplicate work and ensuring consistent responses.

#### Evidence Syncing

The SharePoint integration enables automatic evidence syncing from your SharePoint sites to your Passionfruit Knowledge Base. This allows you to:

- **Select specific sources**: Choose which SharePoint sites, libraries, or folders should be synced with your Knowledge Base
- **Automatic updates**: Evidence files are automatically indexed and updated when changes occur in SharePoint
- **Search and discovery**: Synced evidence becomes searchable within Passionfruit, making it easy to find and reference during questionnaire filling
- **Compliance tracking**: Maintain visibility into evidence sources, ownership, and modification dates directly from SharePoint metadata

#### What is synced

- Selected sites, libraries, or folders and their files.
- Rich metadata used for discovery and compliance (for example: title, type, size, creator/owner, modified/created timestamps, and path).
- File content for search and answer generation where supported (e.g., PDFs, Office documents, and images with text extraction where available).

#### Adding a new company tenant

This section explains how to add a new company tenant to the existing SharePoint app registration and connect it to Passionfruit.

##### Step 1: Allow the tenant in the app registration

Restrict the multi-tenant app registration to specific Entra tenants by adding the new company.

1. Open **Azure Portal** and go to **Microsoft Entra ID**.

<img
  src="/images/integrations/sharepoint-azure-portal-menu.png"
  className="w-full border-none h-auto my-0"
  alt="Azure Portal side menu with Microsoft Entra ID option"
/>

2. Go to **App registrations** and open the SharePoint integration app.

<img
  src="/images/integrations/sharepoint-entra-manage-menu.png"
  className="w-full border-none h-auto my-0"
  alt="Microsoft Entra ID Manage menu with App registrations entry"
/>

<img
  src="/images/integrations/sharepoint-app-registration.png"
  className="w-full border-none h-auto my-0"
  alt="App registrations list showing the SharePoint integration app"
/>

3. Open **Authentication (Preview)** → **Supported accounts**.

<img
  src="/images/integrations/sharepoint-authentication-preview.png"
  className="w-full border-none h-auto my-0"
  alt="App registration Authentication Preview showing Supported accounts section"
/>

4. Click **Manage allowed tenants**.
5. In the dialog, add the new company tenant by **Tenant ID** or domain, then save.

<img
  src="/images/integrations/sharepoint-allowed-tenants.png"
  className="w-full border-none h-auto my-0"
  alt="Manage allowed tenants dialog with the option to add a tenant by ID or domain"
/>

6. Confirm the new tenant appears under **Allowed tenants**.

##### Step 2: Share details with Passionfruit

After your tenant is allowed on the app registration, send the Passionfruit team the information below so we can finish setup on our side. You do not need to run database scripts or call APIs—those steps are handled internally.

| What we need               | Where to find it                                                                                                                         |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| **Tenant ID**              | **Azure Portal** → **Microsoft Entra ID** → **Overview** → **Tenant ID** (also called **Directory (tenant) ID**).                        |
| **SharePoint site URL**    | The full URL of the site that contains the library you want to sync (for example `https://yourcompany.sharepoint.com/sites/Compliance`). |
| **Folder or library path** | The path to the folder or library inside that site (for example `Shared Documents/Policies` or the library name your team uses).         |

If we already shared an **Application (client) ID** with you for this integration, include it when you write in so we can match your organisation to the correct app registration.

Passionfruit configures the integration, connects to your SharePoint scope, and runs the initial sync. We’ll let you know when evidence from SharePoint is available in your Knowledge Base and when ongoing sync is active.
