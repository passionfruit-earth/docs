#### Overview

Passionfruit uses the same pattern as elsewhere: **you can set properties yourself**, and we **add suggestions** for the rest. What you enter manually is **kept**. For fields you leave empty, we propose metadata after upload and can **apply suggestions automatically** when there is nothing to conflict with.

#### Manual first

During upload, assign properties before you finish (for example **Assign entity**, evidence type, or visibility on **Assign properties to all items**). Those values are stored and **not overwritten** later.

1. Start an upload from `Evidence` and add files or links.
2. Use **Assign properties to all items** (or per-file options where available) to set entity, type, and other fields you already know.
3. Complete import. Anything you set here stays as you chose.

#### Suggestions

After processing, Passionfruit runs **auto-tagging**: we suggest type, source, related entities, and more. **Empty** fields get values from these suggestions; we apply them automatically when there is no manual value to preserve. You can still review and change suggested metadata on the evidence detail view.
